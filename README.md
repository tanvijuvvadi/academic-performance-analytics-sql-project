# 🎓 Student Performance Database & Analytics Dashboard

## 📌 Project Overview
This project focuses on designing and analyzing a **Student Performance Management System** using SQL. It simulates a real-world academic environment where institutions track student performance, attendance, course progress, and feedback.

## 🎯 Objective
To build a database system that helps:
- Track student academic performance
- Analyze attendance trends
- Evaluate course effectiveness
- Identify top and at-risk students
- Support data-driven academic decisions

## 🛠️ Tools Used
- SQL (MySQL)

## 🗄️ Database Schema
The project includes the following tables:

- Students  
- Courses  
- Instructors  
- Enrollments  
- Attendance  
- Assessments  
- Marks  
- Feedback  

## 📊 Key Features

### 🔹 Data Generation
- 100+ Students  
- 10+ Courses  
- 10+ Instructors  
- 200+ Enrollments  
- 300+ Attendance Records  
- 200+ Marks Records  
- 100+ Feedback Records  

### 🔹 SQL Analysis

#### Basic Queries
- Fetch all student records
- Course-wise enrollments
- Attendance summary
- Mark's summary

#### Academic Insights
- Top-performing students
- Low-performing students
- Students with low attendance
- Course-wise average marks
- Department-wise performance
- Pass/Fail analysis

#### Advanced Analytics
- Performance classification (CASE)
- At-risk student identification
- Course completion rate
- Instructor-wise performance
- Assessment-wise analysis
- Student ranking (RANK function)
- Improvement tracking

## 📈 Key Insights
- Students with high attendance tend to perform better
- Certain courses have higher completion rates
- Performance varies across departments
- Feedback helps evaluate course quality
